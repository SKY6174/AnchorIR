# 프로젝트 규칙 (Project Rules)

이 프로젝트에서 작업할 때 에이전트는 아래 규칙들을 엄격히 준수해야 합니다.

1. **자동 Git Push 규칙**:
   - 아주 간단한 수정(단순 오타 수정, 사소한 레이아웃 보완 등)이 아닌, 2개 이상의 주요 요구사항이나 기능 수정을 완료하여 정상 작동을 확인하면 자동으로 Git 원격 저장소(`main` 브랜치)에 `git push`를 완료하고 사용자에게 보고할 것.

2. **위원회 의결정족수 및 재적 산정 규칙**:
   - 위원회 성립(의사정족수) 및 안건 의결(의결정족수) 산정 시, 간사(Secretary)는 의결 권한이 없는 행정 실무 진행자이므로 재적 위원 수(`total_quorum`), 출석 의결 위원 수(`attended`), 찬반 표결 정족수 합산 대상에서 엄격히 제외한다.
   - **재적 위원 수 산정**: 총 소속 위원 중 간사를 제외한 의결 위원 수(예: 4명 중 간사 1명 제외 3명)를 해당 위원회의 순수 재적 위원 수로 판정한다.
   - **의사정족수 (성원 요건)**: 간사를 제외한 재적 위원 수(N명)의 과반수(`Math.floor(N / 2) + 1` 명) 이상 출석 시 회원의 성원이 성립된다. (예: N=3명일 경우 2명 이상 출석 시 성원)
   - **의결정족수 (가결 요건)**: 회의 성원 시, 실제 출석(참여)한 위원 수(M명)의 과반수(`Math.floor(M / 2) + 1` 명) 이상이 찬성하면 안건이 최종 가결(의결)된다.

3. **매일 밤 11시 개발일기 및 마일스톤 기록 규칙**:
   - 매일 밤 11시(23:00) 또는 일일 작업 마무리 시점에 그날 수행한 개발 및 마이그레이션(TSX 전환 등)의 주요 내용, 발생한 애로사항(Troubleshooting), 해결 과정, 의사결정 사항을 정리하여 개발일기를 업데이트한다.
   - **`docs/diary/DEV_LOG.md`**: 날짜별 작업 이벤트, 상세 코드 수정 내역, 버그 및 애로사항, 해결 과정을 기록.
   - **`docs/diary/MILESTONE_LOG.md`**: 마일스톤 중심(시스템 초기 구축 ➔ 5대 핵심 모듈 완성 ➔ AI 및 규정 반영 ➔ TSX 마이그레이션 Phase)으로 일자별 변화과정 및 핵심 성과를 자세히 기록.
